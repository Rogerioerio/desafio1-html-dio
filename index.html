Surgimento do HTML

Não é de hojé que a internet no geral, se tornou frequentemente utilizada em pesquisas, montar estratégias, entretenimento, entre outras funcionalidades. E com ela se tornando uma ferramenta requisitada, mas bem nichada por militares e troca de informações, Tim Berners-Lee decidiu criar a WWW, HTML e o primeiro navegador a suportar WWW. E com isso, foi publicado o primeiro site em HTML em 6 de agosto de 1991, sendo info.cern.ch foi o primeiro endereço do primeiro site e servidor web.

"Eu só precisei tomar a ideia de hipertexto e conectá-la às ideias de
Transmission Control Protocol e Domain Name System e - ta-da! - a World Wide Web".
Berners-Lee, Tim. Answers for Young People. World Wide Web Consortium. Consultado em 25 de maio de 2008

O que são Clients?

Clients são dispositivos que estão consumindo dados de um servidor, tanto via internet, tanto via lan, ou algum outro meio, no caso de um servidor web, os dados são requisitados via navegadores. exemplos de Clients: Computadores, Notebooks e Celulares.

O que são Servers?

Servers, como o nome sugere, serve dados para seus Clients, mas também armazenam dados, fazem segurança de requisições, hospedam sites, e etc. Servers são computadores dedicados à estas funções, sendo eles mais caros que os convecionais, por terem peças muito mais poderosas para serem escalaveis. As requisições são feitas pelos navegadores geralmente, retornando os arquivos HTML, que é o que você está vendo do site, CSS, para a estilizar a página HTML, e o Javascript, que realiza certas funções por baixo dos panos do HTML. Os sistemas operacionais de Servers são amplos, tendo Windows Serve, Ubuntu, CentOs, Debian, Fedora e Oracle Linux. E como exemplo de Serves temos:

  • Proxy, Firewall e DNS: Realizam a segurança de requisições.
  • Web Server: Aplicação que não necessita ser instalada, como por exemplo: Youtube.
  • Database e E-mail: Armazenam dados, sendo o E-mail especifico para E-mails e o Database abrangente.

Como é uma estrutura básica de um HTML?

Primeiramente, o HTML não é uma linguagem de programação, ela é um arquivo de marcação, como o CSV, com suas marcações criando sua estrutura chamadas de Tags. Exemplos bons de tags são head e body, sendo head para carregar dados antes de exibir o body que é o corpo da página, onde dentro do body pode ter um texto definido com a tag "p", e com tags de formatação como "h1" para a definição de um texto, "strong" para deixar a letra em negrito e "i" em itálico.

Entendendo a diferença entre listas ordenadas e não ordenadas

Listas ordenadas e listas não ordenadas no HTML são tags que listam elementos dentro delas, sendo uma lista ordenada definada pela tag "ol" e uma lista não ordenada definida por "ul", e os elementos dentro destas listas por "li". A diferença entre as duas, é que a lista ordenada ordena por números, sendo o primeiro item o número "1", o segundo o "2" e assim por diante, enquanto a lista não ordenada mostra cada item com um simbolo de elipse no início de cada um.
